Bootstrap del repositorio Estructura Django, modelos GEI, Flujo Cámara, Biomasa. ✅ Completado.
✓

⚡
Definir responsable técnico por institución Contactar PUJ, IDEAM, UDENAR, UROSARIO para asignar punto de contacto. Desbloquea todas las siguientes tareas.
○
Levantar inventario de fuentes de datos ¿Qué tiene IDEAM? ¿SWAP? ¿Ecopetrol? Formato, cobertura, licencia.

⚡

○
Plan detallado Ene 2026 Cronograma ajustado con responsables confirmados. Kickoff formal de las 4 actividades.
○
Documentar decisiones de arquitectura (ADRs) Registrar stack, estándares y decisiones en la wiki del repo.
Sección 4

Preguntas clave — gaps abiertos

Estas preguntas deben responderse antes o durante los primeros meses del proyecto. Las marcadas en rojo bloquean avance crítico.

📦

Datos existentes

  • ¿Qué datos de carbono tiene IDEAM disponibles, en qué formato y con qué cobertura temporal?
  • ¿Los datos de SWAP están bajo licencia abierta o tienen restricciones de uso y cesión?
  • ¿Qué aporta Ecopetrol al proyecto — datos de flujo, emisiones industriales o financiamiento?
  • ¿Existe ya un inventario de sitios de monitoreo de flujo de cámara activos en Colombia?
  • ¿Los datos de diferentes fuentes usan unidades y nomenclaturas compatibles o requieren homologación profunda?
2 bloqueantes3 importantes
👥

Responsables y gobernanza

  • ¿Quién es el responsable técnico designado en cada institución para el OE2?
  • ¿Quién tiene autoridad para tomar decisiones de arquitectura de datos y estándares?
  • ¿Existe un comité de coordinación interinstitucional con reuniones periódicas?
  • ¿UDENAR tiene un rol técnico específico o es apoyo transversal en todas las actividades?
  • ¿Hay un comité científico definido para la validación de datos (A2.1.10)?
2 bloqueantes3 importantes
⚙️

Técnico y arquitectura

  • ¿La infraestructura cloud está definida (Google CloudSQL) o hay libertad de elegir proveedor?
  • ¿La plataforma debe cumplir lineamientos de datos abiertos del Estado colombiano (Ley 1712)?
  • ¿El bot de WhatsApp (A2.2.10) es un requisito contractual o una propuesta del equipo?
  • ¿Los datos geoespaciales requieren PostGIS o se puede usar un servicio externo (Google Maps, Esri)?
  • ¿Qué nivel de seguridad y autenticación requiere la plataforma — roles por institución, datos públicos/privados?

Contractual y alcance

  • ¿Las fechas del plan de trabajo son compromisos contractuales con el financiador o estimaciones internas?
  • ¿Hay informes de avance trimestrales comprometidos? ¿Quién los elabora?
  • ¿TRL6 es una meta contractual explícita o una aspiración técnica del equipo?
  • ¿Cuáles son los ecosistemas priorizados para la primera versión de la base de datos?
