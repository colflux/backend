Modelo de datos para inventario GEI

Una base Django para registrar gases de efecto invernadero, convertirlos a CO2 equivalente y explorar resultados por territorio, sector, fuente y ano.

CRUD

Gestiona registros de emisiones y catalogos maestros desde vistas Django y el panel admin.

Docker

El entorno incluye Django, Gunicorn y PostgreSQL para una demo reproducible.

Visualizador

Graficas preliminares con agregaciones por ano, sector y gas usando un endpoint JSON.

Entidades principales

EntidadPropositoCampos clave
DatasetMetadataDescribe origen, version, licencia y ano de publicacion del dataset.name, source, version, publication_year
RegionPermite jerarquia territorial: pais, departamento, municipio o cuenca.name, code, region_type, parent
SectorClasifica emisiones segun sector o codigo IPCC.name, ipcc_code, description
GasDefine gas y potencial de calentamiento global a 100 anos.chemical_formula, name, gwp_100
EmissionSourceRepresenta actividades o fuentes emisoras dentro de un sector.name, sector, description
EmissionRecordTabla factual con observaciones anuales y toneladas CO2e.dataset, region, sector, source, gas, year, value_tonnes, co2e_tonnes

Flujo de datos

Los Excel se mapean a catalogos maestros y a la tabla factual EmissionRecord. La plantilla data/excel_mapping_template.csv documenta una convencion inicial para cargar metadatos, territorios, gases, sectores y observaciones.

Como probar

Ejecuta docker compose up --build y abre http://localhost:8000. El comando de arranque carga datos de ejemplo para validar el CRUD y las graficas.
